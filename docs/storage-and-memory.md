# 저장소 / 메모리 구조

## 로컬 저장

- `auto_reply_configs/` : 방별 자동응답 설정 JSON
- `room_state/` : 방별 최근 메시지 JSON
- SharedPreferences : 전역 UI 설정, 가져오기 메타데이터
- `logs/app.log` : 최근 로그

## 메모리 종류

- **사용자 페르소나**: 전역 설정에서 관리
- **방 메모리**: 방별 정책/말투/주의사항 저장
- **최근 대화 흐름**: 실제 수신/발신 메시지 일부 저장
- **가져온 CSV 이력**: 과거 맥락을 방 상태에 병합

## 새 방 자동 누적 규칙

- 사용자가 방을 따로 설정하지 않아도, 새로 관측된 방의 수신 메시지는 기본적으로 `room_state/` 에 누적됩니다.
- 이 누적으로부터 자동 메모리 요약도 함께 갱신됩니다.
- 단, **답장 활성화는 자동으로 켜지지 않습니다.** 새 방은 비활성 room target으로만 등록되고, 사용자가 켜야 실제 자동응답 대상이 됩니다.

현재 구현은 빠른 단순화를 위해 파일 기반 저장을 사용합니다. 대규모 검색/요약이 필요해지면 SQLite/Room으로 확장 가능합니다.
