# AI 공급자 지원 정책

## 현재 구현

- 기본 응답 경로는 **Gemma 4 로컬 답장 엔진**입니다.
- 엔진 입력은 현재 메시지, 최근 방 이력, 방 메모리, 자동 메모리, CSV로 가져온 과거 맥락입니다.
- `ai_judge` / `smart` 모드에서는 확신이 낮으면 답장을 만들지 않고 건너뜁니다.
- 기본 로컬 LLM 다운로드 경로는 **Gemma-4-E2B-it-LiteRT-LM** 입니다.
- Hugging Face 토큰이 있으면 다운로드 요청에 Authorization 헤더를 붙여 gated/private 모델 접근에도 재사용할 수 있습니다.
- 회귀 테스트와 계측 테스트는 다운로드/저장공간 부담을 줄이기 위해 더 작은 **Qwen3-0.6B LiteRT** 테스트 모델을 사용합니다.
- 더 작은 모델은 빠르지만 한국어 품질이 약할 수 있고, 더 큰 모델은 품질이 좋아질 수 있지만 느려질 수 있습니다.

## 지원하지 않는 경로

- 소비자용 ChatGPT Plus/Pro 로그인 연동
- 제3자 앱에서 공급자 계정 세션을 그대로 재사용하는 OAuth 패스스루
- 소비자용 Gemini 유료 플랜 로그인 연동
- 공식 지원 범위를 벗어난 비공식 로그인 연동

## 현재 공급자 정규화

- 저장된 레거시 공급자 값 `openai`, `local`, `local-gguf`, `local-litertlm` 는 읽을 때 모두 `llm` / `gemma-4-e2b-it-litertlm` 으로 정규화합니다.
- 예전 OpenAI API Key, endpoint, authMode 값이 남아 있어도 실행 중에는 버리고 **로컬 Gemma 경로만** 사용합니다.

정상 제품 경로는 네트워크 의존 없이 기기 안의 저장 데이터를 재활용하는 **Gemma 4 로컬 응답 엔진**입니다. LiteRT-LM은 이 경로의 런타임 형식/엔진입니다.

Gemma 4 LiteRT-LM 경로는 현재 **실제 ARM64 Android 기기 검증이 필요**하며, 에뮬레이터에서는 지원하지 않습니다.

로컬 경로에서는 단순한 최근 사실 확인 질문(예: 최근 대화에 이미 나온 시간/일정 확인)에 대해 **모델 생성 전에 최근 대화에서 바로 답을 찾는 빠른 단축 경로**를 함께 사용합니다.

- primary prompt는 페르소나, 방 메모, 최근 대화를 더 넉넉한 예산으로 유지합니다.
- compact prompt는 실패 대비 경로이지만 여전히 **최소 페르소나 + 방 메모 + 최근 대화 4개**를 남깁니다.
- emergency prompt도 0-grounding 으로 떨어지지 않고 **최소 페르소나 + 짧은 방 메모 + 최근 대화 2개**를 유지합니다.

구체적인 최근 측정값은 `docs/testing-quality.md` 의 로컬 수동 계측 결과를 기준으로 봅니다.
