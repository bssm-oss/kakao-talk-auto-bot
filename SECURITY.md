# 보안 정책

## 지원 범위

- 기본적으로 `main` 브랜치와 최신 릴리즈 태그를 우선 지원합니다.
- 오래된 브랜치나 임시 실험 브랜치는 보안 패치 대상에서 제외될 수 있습니다.

## 제보 방법

- 공개 이슈에 취약점을 바로 올리지 마세요.
- 가능하면 GitHub Security Advisory 또는 비공개 채널을 통해 먼저 제보하세요.
- 재현 절차, 영향 범위, 로그, 관련 커밋 정보를 함께 제공해 주세요.

## 제보 시 포함할 정보

- 영향을 받는 버전 또는 커밋
- 재현 단계
- 예상되는 영향도
- 완화 방법이 있다면 그 내용

## 대응 원칙

- 접수 후 재현 가능 여부를 우선 확인합니다.
- 수정이 준비되기 전까지는 공개 토론보다 비공개 조율을 우선합니다.
- 패치 후에는 릴리즈 노트와 관련 문서에 반영합니다.
